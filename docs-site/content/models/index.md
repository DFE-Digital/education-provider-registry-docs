# Data Model Artefacts

This section indexes the formal artefacts that define and validate the Education Provider Registry data model.

The model is split across three layers. The vocabulary and taxonomy define terms and classifications. The ontology defines the structure and relationships. The SHACL shapes express what valid data for each establishment type must look like.

## Artefacts

| Artefact | Format | What it does |
| --- | --- | --- |
| [Vocabulary](../vocabulary/) | SKOS | Defines the terms used across the model - field names, identifiers, concepts |
| [Taxonomy](../taxonomy/) | SKOS | Organises the valid classification values - establishment types, education phases, admissions policies |
| [Ontology](../ontology/) | OWL | Defines the classes and relationships - generated by WIDOCO from the source TTL |
| [Ontology graph viewer](../ontology/webvowl/) | WebVOWL | Interactive force-directed graph of the ontology - explore classes and properties visually |
| [Establishment field rules by type](./establishment-field-rules-by-type/) | Markdown | Complete field list per establishment type - all applicable shapes combined into one view per type |
| [Establishment type field rules](./establishment-type-field-rules/) | Markdown | Per-shape structural rules - which fields are required, optional or not applicable, grouped by SHACL shape |
| [Establishment data quality rules](./establishment-data-quality-rules/) | Markdown | Value-level rules - datatypes, numeric ranges and cardinality constraints for identity, lifecycle and measure fields |
| [SHACL shapes (source)](https://github.com/DFE-Digital/education-provider-registry-docs/blob/main/models/establishment-details-shacl.ttl) | SHACL/Turtle | Machine-readable structural rules - source for the field rules page above |
| [Data quality SHACL shapes (source)](https://github.com/DFE-Digital/education-provider-registry-docs/blob/main/models/establishment-details-data-quality-shacl.ttl) | SHACL/Turtle | Machine-readable value-level rules - source for the data quality rules page above |

## How the layers relate

<ul>
  <li><strong>Vocabulary (SKOS)</strong> - defines what the terms mean</li>
  <li><strong>Taxonomy (SKOS)</strong> - organises the valid classification values</li>
  <li><strong>Ontology (OWL)</strong> - describes the structure and relationships</li>
  <li><strong>SHACL shapes</strong> - expresses per-type validation rules - what valid data must look like</li>
  <li><strong>Field rules (Markdown)</strong> - the same rules in plain English</li>
</ul>

All artefacts share the same namespace (`epr:`) so a term defined in the vocabulary is the same term referenced in the ontology and checked in the SHACL shapes.

## Source files

The Turtle source files are in `models/` in the repository. The vocabulary, taxonomy and field-rules pages on this site are generated or copied from those files during the build.

| File | Description |
| --- | --- |
| `establishment-details-vocabulary-skos.ttl` | SKOS vocabulary source |
| `establishment-details-taxonomy-skos.ttl` | SKOS taxonomy source |
| `establishment-details-conceptual-model.ttl` | OWL ontology source - WIDOCO generates the ontology docs and WebVOWL graph from this file |
| `establishment-details-shacl.ttl` | SHACL shapes source - field rules page is generated from this file during the build |
